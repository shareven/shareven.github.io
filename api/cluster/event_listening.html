  • address <Object>

类似于 cluster.on('listening') 事件，但特定于此工作进程。 

cluster.fork().on('listening', (address) => {
  // 工作进程正在监听。
});

此事件不会在工作进程中触发。

Added in: v0.7.0
  • address <Object>

Similar to the cluster.on('listening') event, but specific to this worker.

cluster.fork().on('listening', (address) => {
  // Worker is listening
});

It is not emitted in the worker.
