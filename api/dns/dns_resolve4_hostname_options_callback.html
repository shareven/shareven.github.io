版本变更
v7.2.0

This method now supports passing options, specifically options.ttl.

v0.1.16

新增于: v0.1.16

  • hostname <string> 需要解析的主机名。

  • options <Object>

    • ttl <boolean> 记录每一条记录的存活次数 (TTL)。当为 true 时，回调会接收一个带有 TTL 秒数记录的类似 { address: '1.2.3.4', ttl: 60 } 对象的数组，而不是字符串的数组。

  • callback <Function>

    • err <Error>
    • addresses <string[]> | <Object[]>

使用 DNS 协议为 hostname 解析 IPv4 地址（A 记录）。 adresses 参数是传给 callback 函数的 IPv4 地址数组（例如：['74.125.79.104', '74.125.79.105', '74.125.79.106']）。

v0.1.16

Added in: v0.1.16

  • hostname <string> Hostname to resolve.

  • options <Object>

    • ttl <boolean> Retrieve the Time-To-Live value (TTL) of each record. When true, the callback receives an array of { address: '1.2.3.4', ttl: 60 } objects rather than an array of strings, with the TTL expressed in seconds.

  • callback <Function>

    • err <Error>
    • addresses <string[]> | <Object[]>

Uses the DNS protocol to resolve a IPv4 addresses (A records) for the hostname. The addresses argument passed to the callback function will contain an array of IPv4 addresses (e.g. ['74.125.79.104', '74.125.79.105', '74.125.79.106']).
